# GDD — 第四关：移动平台

**所属章节**：第一章
**核心机制**：升降台（自触发，驮载玩家）
**场景文件**：`Assets/Scenes/Chapter1/Chapter1_Scene4.unity`

---

## 1. 概述

两个平台之间有一道无法直接跨越的沟壑。沟壑中有一个可移动的平台（自触发 Elevator），初始停在起点侧。玩家滚上平台后，平台自动启动，将玩家水平运送到终点平台，终点 SceneSwitcher 在第二个平台上。核心教学：「站上去，平台会带你走」。

## 2. 玩家感受

「它在动！」——玩家踩上平台的瞬间平台启动，控制权短暂被「借走」，有轻微的失控感和新奇感。到达对岸后控制权归还，玩家立刻恢复自主移动。平台移动过程中无法输入是关键体验节点——玩家第一次感受到机关「主动」影响自己。

## 3. 详细规则

- 布局：起点平台 → 沟壑（中间有移动平台） → 终点平台
- 移动平台：`Elevator`，`selfTriggered = true`，`reset = false`
- `offset` 为水平方向（X 或 Z 轴），距离 = 沟壑跨度
- 玩家滚上平台触发器后，平台携带玩家以 `InOutSine` 缓动水平移动到目标位置
- 移动过程中玩家处于 `ExternalControl` 状态，输入被挂起
- 平台到达终点后 `EndExternalControl`，玩家恢复输入
- 通关条件：玩家从平台走到终点平台，滚入 SceneSwitcher

## 4. 公式

```
平台移动距离 = 沟壑跨度（格数 × 1 单位）
移动时长 moveDuration = 2f 秒
外部控制时长 ≈ moveDuration（DOTween OnComplete 回调归还控制）
```

## 5. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 玩家在平台移动中尝试输入 | `IsExternallyControlled = true`，Player 忽略输入 |
| 玩家在平台到达前滚离平台边缘 | 离开触发器 → riders 列表移除玩家；玩家下落 |
| 平台到达终点后玩家未走向 SceneSwitcher | 平台停在终点位置，玩家恢复自由移动，自行走到终点 |
| 平台已触发时第二个物体踩上 | riders 列表追加，同步携带（本关无第二个可控物体） |

## 6. 依赖

- `Elevator.cs` — 自触发、驮载、offset 移动
- `IExternallyControllable.cs` — 外部控制接口
- `mechanism-elevator.md` — 升降台规则参考
- `Player.cs` — `BeginExternalControl` / `EndExternalControl`
- `SceneSwitcher.cs` — 终点场景加载

## 7. 可调参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| `moveDuration` | 2f 秒 | 平台运送时长；太短失去「被带走」的感受 |
| 沟壑跨度 | 3–5 格 | 太窄感觉不需要平台；太宽等待时间过长 |
| 平台宽度 | 1×1 格 | 与玩家同尺寸，踩上即触发 |

## 8. 验收标准

- [ ] 玩家滚上平台后平台立刻启动，无额外等待
- [ ] 平台运送过程中玩家无法自主移动
- [ ] 平台到达对岸后玩家立刻恢复输入
- [ ] 玩家在运送途中滚离平台会下落（不会被继续携带）
- [ ] 平台停在终点位置不复位（reset = false）
- [ ] 滚入终点触发器后加载 Scene5
