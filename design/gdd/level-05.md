# GDD — 第五关：升降平台 + 移动平台

**所属章节**：第一章
**核心机制**：升降台（顺序依赖）、自触发移动平台、`resetOnArrival` 限时回缩、驮载
**场景文件**：`Assets/Scenes/Chapter1/Chapter1_Scene5.unity`

---

## 1. 概述

沟壑连接两个平台，中间悬空区域有两套独立机关：一节自触发移动平台（开关A，平台本体即触发器）和一块凹陷地面（由场外独立开关 B 控制升起）。两个开关表面上互不相关，实际存在先后依赖——必须先用开关 B 把凹陷地面填平，再踩上移动平台（开关 A），平台才能把玩家送到能够跨过缺口的位置；顺序颠倒时，平台会在抵达后固定等待一段时间后自动回缩，把玩家一并带回起点。

## 2. 玩家感受

「原来顺序不对」——玩家踩错顺序后，移动平台把自己送到终点附近却过不去，短暂等待后平台又把自己原样带回起点。这不是文字提示告诉玩家的，而是亲身经历的后果：不做惩罚（无伤亡、无重来关卡），只是白等一趟，逼玩家反推「这条路走不通，得先干别的」。移动平台与凹陷地面之间是真实的先后依赖，不是两次互不影响的独立触发。

## 3. 详细规则

- 布局：起点平台 → 沟壑（中段悬空，内含移动平台 + 一块凹陷地面块） → 终点平台
- 开关 A（移动平台本体）：`Elevator`，`selfTriggered = true`，`reset = true`，`resetOnArrival = true`，`offset` 水平指向终点方向
  - 玩家踩上其自身触发器立即出发，无需等待
  - 抵达目标位置后固定 `resetDelay` 秒后开始回缩，不论玩家是否仍站在平台上；回缩时通过 `CarryRiders(-offset)` 把仍在平台上的玩家一并带回起点
- 开关 B（独立地面开关）：场外独立 `ElevatorSwitch`，`holdDuration` 默认 1f 秒，目标是凹陷地面块所在的 `Elevator`：`selfTriggered = false`、`reset = false`、`offset` 向上使其与两侧地面齐平
  - 一旦升起即永久保持（`reset = false`），不会再缩回
- 正确顺序：先触发开关 B（凹陷地面永久升起，缺口被填平）→ 再踩上移动平台（开关 A）→ 平台把玩家送到缺口边缘 → 缺口已填平，直接跨过到终点
- 错误顺序：先踩移动平台（开关 A）→ 平台出发，把玩家送到接近终点平台的位置，但缺口仍未填平，过不去 → `resetDelay` 秒后平台自动回缩，把玩家一并带回起点 → 玩家重新尝试，这次改为先触发开关 B

## 4. 公式

```
resetDelay（开关A所在Elevator） = 3f 秒（建议值，与 mechanism-elevator.md 默认一致）
moveDuration = 2f 秒（两个 Elevator 共用缓动时长）
holdDuration（开关B） = 1f 秒（默认）
正确解触发次数 = 2（先 B 后 A）
错误顺序的等待代价 ≈ resetDelay + moveDuration × 2 秒（送出 + 回缩）
```

## 5. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 先踩 A（移动平台）再触发 B | 平台已回缩带玩家回起点，需要重新踩 A；此时 B 已生效、缺口已填平，第二次踩 A 可以直接通过 |
| 玩家在平台移动途中触发 B | B 的地面升起与 A 的平台运送是两个独立机关，互不干扰；只影响 A 抵达后玩家能否跨过缺口 |
| 玩家在平台回缩过程中试图脱离 | 携带期间 `isExternallyControlled = true`，输入被挂起，无法主动离开 |
| 玩家踩开关 B 后中途离开触发器，且未满 `holdDuration` | `ElevatorSwitch.OnTriggerExit` 取消待触发的协程，凹陷地面不升起 |
| 玩家踩开关 B 满 `holdDuration` 后才离开 | `TriggerMove()` 已调用，凹陷地面开始升起，离开触发器不影响已发起的动画（不可逆） |

## 6. 依赖

- **Elevator.cs** — `resetOnArrival` 限时回缩模式（开关A）、`reset = false` 永久模式（开关B目标）
- **ElevatorSwitch.cs** — 独立开关（开关B）
- **IExternallyControllable.cs** — 驮载接口
- **Player.cs** — `BeginExternalControl` / `EndExternalControl`
- **mechanism-elevator.md** — 升降台规则参考

## 7. 可调参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| `resetDelay`（开关A） | 3f 秒 | 错误顺序下玩家等待多久才被带回；太短玩家来不及意识到过不去，太长显得拖沓 |
| `moveDuration` | 2f 秒 | 两个机关共用的运送/升起节奏 |
| `holdDuration`（开关B） | 1f 秒 | 触发凹陷地面升起前的踩踏确认时间 |
| 沟壑跨度 | 2–4 格 | 决定移动平台单次运送距离，过宽会拉长错误顺序的等待成本 |

## 8. 验收标准

- [ ] 先踩移动平台（不碰开关B）：平台出发，抵达后等待 `resetDelay` 秒自动缩回并带回玩家
- [ ] 先触发开关B（凹陷地面升起）：缺口与两侧地面完全齐平
- [ ] 正确顺序（先B后A）：移动平台抵达后玩家可直接跨过缺口到终点，不会被强制带回
- [ ] 错误顺序不构成惩罚性失败（无需重启场景），允许玩家立即重新尝试
- [ ] 两个开关相互独立触发，互不干扰对方状态
- [ ] 滚入终点触发器后加载 Scene6
