# GDD — 第一关：基本翻滚

**所属章节**：第一章
**核心机制**：滚动移动
**场景文件**：`Assets/Scenes/Chapter1/Chapter1_Scene1.unity`

---

## 1. 概述

第一关是零压力教学关，布局为一条从左到右的直线平台，无高度差、无机关、无障碍。玩家只需按方向键将立方体滚到终点触发器即可通关。目标是建立「按键 → 滚动一格」的操作直觉，并感受 InOutSine 缓动带来的滚动手感。

## 2. 玩家感受

「原来这就是滚动的感觉」——玩家第一次按键后，立方体以干净的缓动翻滚一格，立刻明白了操控单位和响应方式。没有失败压力，只有探索和确认。

## 3. 详细规则

- 平台：单条直线，长度 N 格（建议 6–8 格），宽 1 格
- 起点：玩家出生在左端
- 终点：SceneSwitcher 触发器放置在右端最后一格
- 通关条件：玩家滚入 SceneSwitcher 触发器，加载 Scene2
- 无任何障碍、高度差或机关
- 玩家可以向左滚出起点边缘掉落（当前无复位机制，需重启场景）

## 4. 公式

```
平台总长度 = N 格（N 建议 6–8）
每格边长 = 1 单位（cubeHalfSize × 2 = 1.0f）
最短通关步数 = N - 1 步
```

## 5. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 玩家向左滚出平台 | 触发下落；当前无复位机制，玩家需重启场景 |
| 快速连按方向键 | Player.cs 每次只处理一个移动协程，多余输入被忽略 |
| 玩家向上/下方向滚 | 宽 1 格，立刻离开平台触发下落 |

## 6. 依赖

- `Player.cs` — 滚动移动、输入处理
- `SceneSwitcher.cs` — 终点触发与场景加载
- 无机关依赖

## 7. 可调参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| 平台长度 N | 6–8 格 | 通关时长；太短失去教学感，太长显得无聊 |
| `Player.rollDuration` | 0.25f | 滚动手感；本关唯一需要感受的核心参数 |

## 8. 验收标准

- [ ] 按右键，立方体向右滚动恰好一格，动画流畅
- [ ] 连续按键可以连续滚动，无卡顿或跳格
- [ ] 滚入终点触发器后自动加载 Scene2
- [ ] 向左滚出平台边缘后立方体正常下落（不卡在空中）
- [ ] 关卡内无任何多余碰撞体或触发器干扰移动
