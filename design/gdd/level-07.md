# GDD — 第七关：推箱子（填沟）

**所属章节**：第一章
**核心机制**：推箱子（PushableBlock）、下落
**场景文件**：`Assets/Scenes/Chapter1/Chapter1_Scene7.unity`

---

## 1. 概述

沟壑跨度较宽，超过单个箱子能一次性填平的宽度，需要把两个箱子先后推入沟中才能拼出完整的可通行路面。两个箱子分别停在起点区域两侧、朝向不同，玩家必须先推第一个箱子入沟，再绕到另一侧对齐第二个箱子的推动方向，才能把它也推进沟里补齐缺口。

## 2. 玩家感受

「这次一个不够」——推入第一个箱子后走到沟边，发现还差一截过不去，意识到场上还有第二个箱子；绕到侧面对齐推动方向的过程带来「路线规划」而非单纯「推箱子」的体验。与第六关刻意形成对比：第六关两次推箱子的动作完全相同、变化的是地形；本关两次推箱子的路线和朝向不同，是接力而非重复。

## 3. 详细规则

- 布局：起点区域两侧各放置一个 `PushableBlock`，推动方向不同——例如一个正对沟壑可直接推入，另一个需要玩家先绕到侧面调整站位才能对准同一道沟壑
- 沟壑跨度：建议 2 格，单个箱子（1×1×1）推入只能填平其中 1 格，必须两个箱子先后填入相邻的两格才能形成完整可通行路面
- 箱子跌落后能否「垫脚」取决于沟底深度是否恰好等于 1 个单位（箱子高度）：`PushableBlock.FallIfUnsupported` 触发后箱子转为非运动学，交由物理引擎接管下落，停在物理结算的位置，不会额外吸附网格——因此沟底深度必须与箱子高度精确匹配，箱子跌落后顶面才会与两侧地面齐平，形成可行走的桥面；深度不匹配会导致箱子陷得过深（留缺口）或凸出过高（变成新的墙）
- 通关条件：两个箱子都被推入沟壑并各自补齐一格后，沟壑整体与两侧地面齐平，玩家从起点滚过完整路面到达终点 `SceneSwitcher`

## 4. 公式

```
沟壑跨度 = 2 格（建议值，需等于待填入箱子数量，否则无法完全填平）
箱子跌落垫脚有效条件：沟底深度 = cubeHalfSize × 2（即 1 个单位，箱子完整高度）
推动动画时长 pushDuration ≈ Player.rollDuration（0.25f），确保两次推箱手感与滚动一致
```

## 5. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 两个箱子被推向同一格 | `IsOccupied` 检测到目标格已被第一个箱子占用，第二次推动失败——布局阶段必须确保两个箱子分别对应沟壑中不同的格 |
| 沟底深度与箱子高度不匹配 | 箱子陷入过深或凸出过高，通关路径出现台阶或空隙——关卡搭建阶段需用射线/网格核实沟底 Y 坐标 |
| 玩家先推第二个箱子、再推第一个 | 两个箱子相互独立，允许任意顺序推入，无强制先后依赖（与第五关的强制顺序不同） |
| 箱子推入沟壑后未与另一格完全对齐 | 两格之间留下缝隙，玩家滚过时因缝隙处无支撑触发下落——搭建阶段需核对两个目标格是否真正相邻 |

## 6. 依赖

## 7. 可调参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| 沟壑跨度 | 2 格 | 必须等于场上可推入箱子数量，否则无法完全填平 |
| 两个箱子的初始朝向/位置 | 起点两侧，方向不同 | 决定玩家是否需要绕路才能对齐推动方向；差异越大寻路难度越高 |
| 沟底深度 | 精确等于 1 单位（箱子高度） | 决定箱子跌落后能否与地面齐平形成桥面 |

## 8. 验收标准

- [ ] 推动任意一个箱子入沟，该箱子精确落入并补齐对应的一格
- [ ] 两个箱子都推入后，沟壑整体与两侧地面完全齐平，无缝隙/无凸起
- [ ] 只推入一个箱子时沟壑仍有缺口，玩家尝试跨过会因无支撑而下落
- [ ] 两个箱子的推动顺序不影响最终结果（无强制先后依赖）
- [ ] 玩家绕到侧面对齐第二个箱子的推动方向后可以正常推动（未被路径几何卡死）
- [ ] 滚入终点触发器后加载 Scene8
