# GDD — 第六关：推箱子（丘陵地形，两次开路）

**所属章节**：第一章
**核心机制**：推箱子（PushableBlock）、地形起伏挡路（垫高地块，无新增脚本）
**场景文件**：`Assets/Scenes/Chapter1/Chapter1_Scene6.unity`

---

## 1. 概述

原本开阔的平地被改造为高低起伏的小土丘。地形本身不新增任何脚本机关，纯靠垫高部分地块（哪怕只垫高 0.25 单位，翻滚判定就会视为撞墙）让绝大多数格子变得不可通行，只留一条先向南、再转向东的蜿蜒小路。路径中段被一个新增的 `PushableBlock3` 挡住，推开后才能继续前进，走到本关最初版本（单箱设计）就预留的窄通道，通道尽头是第一个箱子，同样需要推开才能到达终点开关。

## 2. 玩家感受

「一马平川」变成「看得到却走不过去」——大部分地面在视觉上仍是平坦的方块拼接，但只有沿路的几格真正可站。玩家需要靠试探找出唯一的窄路，找到路后又在中段被箱子第二次拦下。两次推箱子的动作完全相同（没有引入新规则），变化的是地形本身逼出的路线感——目的是在「单一机制」内先做一次内容量的扩展，为第七关「双箱接力」的路线规划难度做铺垫。

## 3. 详细规则

- 坐标系：以起点所在格为 (0,0)，x 向东为正，z 向北为正，沿用既有地形网格
- 路径（唯一可通行格，均保持原始 y = -0.5）：(0,0) → (0,-1) → (0,-2) → (1,-2) → (2,-2) → (3,-2)[箱子挡路] → (4,-2)，即先向南走三格，再转向东走三格
- 垫高地块（共 18 格，原始 y = -0.5，全部抬高至少 0.25 单位形成阻挡）：
  - z=0 行：(1,0)→0，(2,0)→0.25，(3,0)→0.5，(4,0)→0.25
  - z=-1 行：(1,-1)→0.25，(2,-1)→0.5，(3,-1)→0.25，(4,-1)→0
  - z=-3 行：(0,-3)→0，(1,-3)→0.25，(2,-3)→0.5，(3,-3)→0.25，(4,-3)→0
  - z=-4 行：(0,-4)→0.25，(1,-4)→0.5，(2,-4)→0.25，(3,-4)→0，(4,-4)→-0.25
- 第二个箱子 `PushableBlock3`：放置在路径中段 (3,-2)，玩家沿路径走到此处会被完全挡住，须向东推开一格（推到 (4,-2)）才能继续前进
- 通道尽头的第一个箱子（初版单箱设计遗留）：位置与逻辑不变，推开后到达终点开关
- 撞墙判定原理：`Player.GetBlockingCollider` 在玩家**当前** Y 高度做 `OverlapBox` 检测；每块 `CubicTerrain` 的 `BoxCollider` 都是完整的 1.0 单位高，只要相邻格子被垫高（哪怕只有 0.25），就会在玩家当前高度产生碰撞体重叠，判定为墙——由于攀爬功能当前被禁用，玩家无法翻滚上垫高地块，因此垫高地块无需做成陡峭悬崖，最小 0.25 的抬升即可形成有效阻挡
- 反之，若某格被**降低**，翻滚检测在玩家当前高度不会重叠（判定可通行），玩家滚上去后 `FinishAfterRoll` 发现脚下没有支撑会转为下落——这是本关未使用、但同一套判定系统支持的另一种地形变化方式

## 4. 公式

```
最小有效阻挡高度差 = +0.25（相对玩家当前 Y）
垫高地块判定盒 = OverlapBox(center=(x, 玩家当前Y, z), halfExtents=cubeHalfSize × 0.9)
路径长度 = 6 步（南 3 + 东 3），含 1 次推箱
```

## 5. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 玩家尝试滚上任意垫高地块 | `GetBlockingCollider` 命中该地块碰撞体，触发 `ShakeFeedback`，原地不动 |
| 玩家推动 `PushableBlock3` | 目标格 (4,-2) 为空地，`IsOccupied` 检测通过，推动正常成功 |
| 玩家尝试绕开既定路径接近终点 | 除既定路径 7 格外其余 18 格全部不可通行，不存在绕路可能 |
| 玩家推箱子方向偏离路径 | 箱子只在路径方向上被规划为可推（垫高地块与推动方向不冲突），无需额外限制 |

## 6. 依赖

- **CubicTerrain 预制体的 BoxCollider**（完整 1.0 单位高）— 撞墙判定的物理基础
- **Player.cs** — `GetBlockingCollider` / `FinishAfterRoll`（下落判定）
- **PushableBlock.cs** — 新增的 `PushableBlock3`
- **mechanism-pushable-block.md** — 推箱子规则参考

## 7. 可调参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| 垫高地块的 y 值 | 0 / 0.25 / 0.5 三档 | 制造起伏感的视觉高度；理论上只需 +0.25 即可形成阻挡，更高只是视觉效果 |
| 路径宽度 | 单格宽，唯一路径 | 决定寻路难度；加宽会削弱「只有一条路」的解谜感 |
| 箱子位置 (3,-2) | — | 决定推箱子发生在路径的哪一步 |

## 8. 验收标准

- [ ] 除既定路径 7 格外，其余 18 格垫高地块均无法翻滚上去，触发 `ShakeFeedback`
- [ ] 沿路径从起点走到 (2,-2) 后被 `PushableBlock3` 完全挡住，必须推开才能继续
- [ ] 推开 `PushableBlock3` 后可以继续走到 (4,-2) 并进入原窄通道
- [ ] 窄通道尽头的第一个箱子仍可正常推开到达终点
- [ ] 场景内无浮空/穿模的地形块（垫高操作后地块底部仍与相邻地块自然衔接）
- [ ] Console 无编译错误或运行时报错（本次改动已通过 `read_console` 检查）
