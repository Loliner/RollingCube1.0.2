# GDD — 第三关：升降桥

**所属章节**：第一章
**核心机制**：升降台、外部开关（ElevatorSwitch）
**场景文件**：`Assets/Scenes/Chapter1/Chapter1_Scene3.unity`

---

## 1. 概述

两个平台之间有一道沟壑，中间有一块地面（桥体）初始处于低位。玩家站在第一个平台上的开关（ElevatorSwitch）一段时间后，桥体升起与两侧平台齐平，玩家可以走过去到达终点。核心教学：「踩住开关才能触发机关」。

## 2. 玩家感受

「我踩住这个地方，什么东西动了」——玩家在开关上等待，看到沟壑中间的地面缓缓升起，直觉上明白了「开关 = 触发」的概念，并体验到延迟触发带来的期待感。

## 3. 详细规则

- 布局：起点平台 → 沟壑（中间有桥体） → 终点平台
- 开关（ElevatorSwitch）放在起点平台靠近沟壑的位置
- 玩家进入开关触发器并持续停留 `holdDuration` 秒后，桥体升起
- 提前离开开关则取消触发，桥体不动
- 桥体：`Elevator`，`offset` 向上，`reset = false`（升起后永久保持）
- 桥体升起后高度与两侧平台齐平，玩家可直接滚过
- 通关条件：滚入终点平台的 SceneSwitcher

## 4. 公式

```
桥体上升距离 = 沟壑深度（使桥面与平台面等高）
开关持续时间 holdDuration = 1f 秒（默认）
桥体移动时长 moveDuration = 2f 秒（默认）
```

## 5. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 玩家踩到开关后立刻离开 | 触发取消，桥体不动 |
| 桥体上升时玩家站在桥上 | 桥是纯地形，非 selfTriggered，玩家不会被携带 |
| 桥升起后玩家再次踩开关 | `isTriggered = true`，`TriggerMove` 无效，不重复触发 |
| 玩家跳过开关直接尝试过沟 | 沟壑宽度 > 1 格，无法直接滚过，被迫使用开关 |

## 6. 依赖

- `ElevatorSwitch.cs` — 开关逻辑（`holdDuration`、多目标触发）
- `Elevator.cs` — 桥体升降（`offset`、`reset = false`）
- `mechanism-elevator.md` — 升降台规则参考
- `Player.cs` — 触发检测、移动
- `SceneSwitcher.cs` — 终点场景加载

## 7. 可调参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| `holdDuration`（开关） | 1f 秒 | 太短玩家误触；太长体验拖沓 |
| `moveDuration`（桥体） | 2f 秒 | 桥升起的视觉节奏 |
| 沟壑宽度 | 2–3 格 | 确保玩家无法直接跳过，必须触发桥 |

## 8. 验收标准

- [ ] 踩住开关 1 秒后桥体开始上升，动画流畅
- [ ] 未满 1 秒离开开关，桥体不动
- [ ] 桥升起后与两侧平台完全齐平，玩家可无缝滚过
- [ ] 桥升起后再踩开关不会重复触发或抖动
- [ ] 滚入终点触发器后加载 Scene4
