# GDD — 第二关：立体世界

**所属章节**：第一章
**核心机制**：滚动移动、高度差下坡
**场景文件**：`Assets/Scenes/Chapter1/Chapter1_Scene2.unity`

---

## 1. 概述

第二关引入高度差概念。平台分为高台和低台两段，中间有一到两级台阶（0.5 单位高度差）。玩家必须先向前滚动，翻过台阶下坡，才能抵达终点。目标是让玩家理解「立方体可以在三维空间里移动，高度变化是正常的」。

## 2. 玩家感受

「哇，还可以下台阶」——玩家滚到台阶边缘时，立方体自然翻下，落地继续滚动。体验从平面延伸到立体，空间感初步建立。

## 3. 详细规则

- 布局：两段平台，高台在起点侧，低台在终点侧
- 台阶高度差：0.5 单位（半格，与 Scene2 实际搭建一致）
- 台阶数量：1–2 级
- 通关条件：滚入低台终点的 SceneSwitcher
- 玩家从高台向低台方向滚时，落地由 Player.cs 的下落逻辑处理（`StartFalling` → `LandWhenSettled`），落点 Y 轴取射线碰撞点 + `cubeHalfSize`
- 无机关，无推箱子

## 4. 公式

```
台阶高度差 = 0.5 单位（cubeHalfSize × 1）
落地 Y = hit.point.y + cubeHalfSize（由 Player.LandWhenSettled 计算）
最短通关步数 = 高台段格数 + 低台段格数
```

## 5. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 玩家从低台尝试向上滚回高台 | 被台阶壁面阻挡，触发 ShakeFeedback |
| 玩家下台阶时旋转未对齐 | LandWhenSettled 落地后强制吸附最近 90° |
| 玩家滚出两侧边缘 | 下落，当前无复位 |

## 6. 依赖

- `Player.cs` — 下落逻辑（`StartFalling` / `LandWhenSettled`）、`ShakeFeedback`
- `SceneSwitcher.cs` — 终点场景加载

## 7. 可调参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| 台阶高度差 | 0.5 单位 | 低于 0.25 视觉效果不明显；高于 1.0 超出单次下落范围 |
| 台阶数量 | 1–2 级 | 超过 2 级会让教学复杂度跳升 |
| 高/低台各自长度 | 各 4–6 格 | 保证玩家有足够空间建立方向感 |

## 8. 验收标准

- [ ] 玩家向台阶方向滚动后自然下落，落地位置准确
- [ ] 落地后立方体旋转与地面对齐（无歪斜）
- [ ] 从低台尝试向上滚被阻挡，触发抖动反馈
- [ ] 滚入终点触发器后加载 Scene3
- [ ] 无穿模或卡在台阶边缘的情况
